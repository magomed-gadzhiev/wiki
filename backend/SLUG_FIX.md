# Исправление проблемы с полем slug

## Проблема
При создании статьи через API возникала ошибка:
```
slug: ["Обязательное поле."]
```

## Причина
Поле `slug` в модели `Article` было обязательным, но должно было генерироваться автоматически из заголовка.

## Решение

### 1. Изменена модель Article
- Добавлено `blank=True` к полю `slug` в модели
- Поле теперь может быть пустым при создании

### 2. Обновлен сериализатор ArticleSerializer
- Поле `slug` сделано необязательным через `extra_kwargs`
- Добавлена автоматическая генерация slug в методах `create()` и `update()`

### 3. Создана миграция базы данных
- Миграция обновляет структуру таблицы для поддержки `blank=True`

## Как это работает

При создании статьи:
- Если `slug` не указан, он автоматически генерируется из `title`
- Если `slug` указан явно, используется указанное значение

При обновлении статьи:
- Если изменяется `title` и `slug` не указан, slug обновляется автоматически
- Если `slug` указан явно, используется указанное значение

## Пример использования API

**Создание статьи без slug:**
```json
{
  "title": "Первая статья",
  "content": "<p>Содержимое</p>",
  "summary": "Описание"
}
```
Slug будет автоматически создан как "pervaya-statya"

**Создание статьи с явным slug:**
```json
{
  "title": "Первая статья",
  "slug": "my-custom-slug",
  "content": "<p>Содержимое</p>"
}
```
