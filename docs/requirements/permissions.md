# Требования к системе прав доступа

## Система групп и прав на категории

### Группы пользователей
- Один пользователь может быть в нескольких группах
- Группы создаются и управляются администраторами
- Пользователи могут быть добавлены в группы через Django Admin или API

### Права групп на категории
Каждая группа может иметь один из трех уровней доступа к категории статей:

1. **Нет доступа (none)**
   - Группа не имеет доступа к статьям данной категории
   - Пользователи группы не увидят статьи этой категории
   - **Создание статей в категории запрещено** пользователям без доступа

2. **Только чтение (read)**
   - Группа может только читать опубликованные статьи категории
   - **Черновики не видны** пользователям с правом только на чтение
   - Редактирование и удаление запрещены
   - **Создание статей в категории запрещено** пользователям с правом только на чтение
   - **Просмотр истории версий запрещен** пользователям с правом только на чтение

3. **Полные права (full)**
   - Группа имеет полный доступ к статьям категории
   - Может читать все статьи (включая черновики)
   - Может редактировать и удалять статьи

## Уровни доступа (старая система - для обратной совместимости)

### Просмотр (can_view)
- Пользователи с правом просмотра могут читать статью
- Автор статьи автоматически имеет право просмотра
- Администраторы имеют все права

### Редактирование (can_edit)
- Пользователи с правом редактирования могут изменять содержимое статьи
- Автор статьи автоматически имеет право редактирования
- При редактировании создается новая версия статьи

### Удаление (can_delete)
- Пользователи с правом удаления могут удалять статью
- Автор статьи автоматически имеет право удаления
- Удаление необратимо (но версии сохраняются в БД)

## Правила доступа

### Приоритет проверки прав:
1. **Суперпользователи (is_superuser)** - всегда имеют полный доступ ко всем статьям, независимо от других настроек
2. **Автор статьи** - имеет все права на свою статью
3. **Права через группы на категорию** - проверяются права всех групп пользователя на категорию статьи
4. **Индивидуальные права** (can_view, can_edit, can_delete) - используются для статей без категории или если нет прав через группы

### Для статей с категорией:
- Если у пользователя есть права через группы на категорию:
  - **Полные права (full)**: видит все статьи категории (включая черновики), может создавать, редактировать и удалять
  - **Только чтение (read)**: видит только опубликованные статьи категории, черновики не видны, создание статей запрещено, просмотр истории версий запрещен
  - **Нет доступа (none)**: статьи категории не видны, создание статей запрещено
- Если у пользователя нет прав через группы, используются индивидуальные права (can_view, can_edit, can_delete)

### Для статей без категории:
- Используются индивидуальные права (can_view, can_edit, can_delete)
- Публичные статьи видны всем авторизованным пользователям
- Черновики видны только автору и пользователям с правом can_view

### Публичные статьи (is_published=true)
- Видны всем авторизованным пользователям (если нет ограничений через группы)
- Редактирование и удаление только для автора и пользователей с соответствующими правами

### Черновики (is_published=false)
- Видны только автору и пользователям с полными правами через группы
- При праве только на чтение через группы - черновики не видны
- Редактирование и удаление только для автора и пользователей с соответствующими правами

## Реализация

- Права проверяются на уровне бэкенда через `ArticlePermission`
- При создании статьи проверяются права на категорию: пользователи с правом только на чтение (read) или без доступа (none) не могут создавать статьи в категории
- При просмотре истории версий проверяются права на категорию: только пользователи с полными правами (full), авторы и суперпользователи могут просматривать версии
- Фильтрация статей в списке учитывает права групп на категории
- На фронтенде отображаются кнопки действий только для пользователей с соответствующими правами
- На фронтенде ссылка на историю версий скрыта для пользователей с правами только на чтение (read)
- API возвращает ошибку 403 при попытке доступа без прав

## API для управления группами и правами

- `GET /api/articles/groups/` - список групп
- `POST /api/articles/groups/` - создание группы
- `GET /api/articles/groups/{id}/` - детали группы
- `PUT/PATCH /api/articles/groups/{id}/` - обновление группы
- `DELETE /api/articles/groups/{id}/` - удаление группы

- `GET /api/articles/category-permissions/` - список прав на категории
- `POST /api/articles/category-permissions/` - создание права
- `GET /api/articles/category-permissions/{id}/` - детали права
- `PUT/PATCH /api/articles/category-permissions/{id}/` - обновление права
- `DELETE /api/articles/category-permissions/{id}/` - удаление права
