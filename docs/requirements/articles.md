# Требования к функционалу статей

## Основные возможности

### Создание и редактирование статей
- Создание новых статей с заголовком, кратким описанием и содержимым
- Редактирование существующих статей
- Поддержка визуального редактора GrapesJS с возможностью:
  - Визуальное редактирование с drag-and-drop интерфейсом
  - Форматирование текста (жирный, курсив, подчеркивание)
  - Заголовки разных уровней
  - Списки (нумерованные и маркированные)
  - Вставка изображений через Asset Manager
  - Вставка таблиц и layout блоков Bootstrap
  - Ссылки
  - Цвета текста и фона
  - Использование Bootstrap компонентов (контейнеры, строки, колонки)
  - Сохранение стилей и форматирования

### Импорт из Word
- Загрузка .doc и .docx файлов
- Автоматическая конвертация содержимого в HTML через Pandoc
- Fallback на mammoth если Pandoc недоступен
- Сохранение форматирования (параграфы, таблицы, стили, layout)
- Конвертация элементов в Bootstrap компоненты
- Сохранение нестандартных стилей через inline-стили
- Вставка конвертированного содержимого в редактор GrapesJS

### Изображения
- Загрузка изображений для статей
- Отображение изображений в галерее
- Альтернативный текст для изображений
- Автоматическая генерация URL для доступа к изображениям

### Публикация
- Статьи могут быть опубликованы или находиться в черновике
- Публикованные статьи видны всем авторизованным пользователям
- Черновики видны только автору и пользователям с правами просмотра

### Комментарии
- Любой пользователь с правами на чтение (read) или редактирование (edit) может оставлять комментарии к статьям
- Комментировать можно только опубликованные статьи
- Комментарии создаются с использованием wysiwyg редактора Quill с поддержкой форматирования текста
- Пользователи могут отвечать на комментарии (вложенные комментарии)
- При создании комментария или ответа можно ссылаться на другие комментарии
- Автор комментария может удалять свои комментарии
- Комментарии отображаются в иерархическом виде с поддержкой вложенности

### Графики по CSV
- Возможность добавления графиков в содержимое статьи через визуальный редактор
- Графики создаются на основе данных из CSV файлов
- Каждый график загружается из отдельного CSV файла с двумя колонками (X и Y координаты)
- На одной координатной сетке могут размещаться несколько графиков
- При добавлении блока графика открывается диалоговое окно для настройки:
  - Название графика
  - Подписи осей X и Y
  - Диапазоны значений для осей (опционально)
  - Для каждого графика:
    - Название серии
    - Цвет линии
    - Толщина линии
    - Стиль линии (сплошная, пунктирная, точечная)
    - Загрузка CSV файла с координатами
- CSV файлы парсятся на клиенте, данные сохраняются в JSON формате в контент статьи
- Графики отображаются с использованием библиотеки Plotly.js
- Поддержка интерактивных элементов: подсказки при наведении, легенда, масштабирование, перемещение графика
